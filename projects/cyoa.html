CYOA Project

Context

Before this project, we learned how to use github, or IDE as well as how to link files which were all core ideas needed for our next project, CYOA project or Create Your Own Adventure. The concept of the project was to use those links to bring the viewer from file A to file B with them choosing what they did in which scenerio ultimately allowing the viewer to experience an adventure we made. This project was hyped up by our teacher as "The Best Project" and other then the coding being a bit annoying it was pretty entertaining especially since we got to pick our adventure.

Content

What scenario we got and who we worked with was worked in an awkward way. At first, we got to list certain scenerios we would like to do then our teacher made partnerships that made sure we got our top5 that we shared with another classmate, making us partners. In our project about, Gambling Money, we sent you to a casino letting you play at the Roulette Wheel(My side) and Texas Hold Em (My partner's side). We first planned out our ideas using Google Drawing then using our IDE, worked on the projcet.

Challenges

This project wasn't too difficult, markdown linking wasn't anything troublesome and other then some errors with pulling and pushing with our IDE, we got it done kinda simple. We were able to finishwith a few days to spare therefore me and my partner decided to go beyond and do the possible extra credit, making our existing markdown project into html. Html was brutal, the linking was different, we had to make entirely new folders and files and though we had our core ideas on our markdown made project, html is just more complicated. For instance, to link html, we had to use the href tag insteading of just using `[text](link)`. We also tried to use CSS which was a pain because of how it worked. We had to copy and paste a certain message to every single HTML file we had to link those html's to our css. We ran into many errors like it not properly showcasing because we had accidently made an index.html folder from a merge that we completely forgot about. We ended up getting a lot of help from our teacher but ultimately were able to finish before the deadline.

Takeaways

Some takeaways I had about my project was to not overcomplicate things especially since we decided to do the extra credit. This was too pointed out by our teacher how if we invested too much with the detail we wouldn't be able to get it done. At first, I was going to overcomplicate things by adding a lot more scenarios that could've happen instead of reusing them but looking back, we barely scraped by on finishing the project especially since HTML was a lot more complicated and would've made things a lot harder on myself even if it was of an interesting idea.If I was to in the future make something similair, I definetly would make sure the planning was more polished because it was very important on making sure the project can flow well..

Next Step

The project itself is definetly not finished design wise as it lacks lot of elements of css but overall it was a nice product. If im being completely honest, I wasn't fond of how things came out since I had alot more ideas but I do wish it was easier to pour my ideas into the project but because it has to flow with everything else, it more like adding a new cog then adding another card to the deck.
